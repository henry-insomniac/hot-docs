# Bug 修复工作流

## 适用场景
修复已知 Bug（路径问题、插件加载失败、内存泄漏）

## 工作流程

### 1. 创建修复分支
```bash
/branch 基于 Issue #XX 创建修复分支
```
- 命名格式：`fix/<scope>/<issue-number>-<description>`

### 2. 问题诊断
```bash
/debug 诊断问题根因
```
- 复现问题
- 收集信息
- 定位根因

### 3. 编写测试（先测试后修复）
```bash
/test 编写复现 Bug 的测试用例
```
- 测试应该失败（验证 Bug 存在）
- 覆盖边界情况

### 4. 修复实现
```bash
/implement 修复 Bug
```
- 添加防御性代码
- 验证测试通过

### 5. 回归测试
```bash
/test 补充回归测试
```
- 确保修复不引入新问题

### 6. 代码提交
```bash
/commit 提交修复代码
```
- 格式：`fix(<scope>): 描述，closes #XX`

### 7. 创建 PR
```bash
/pr 创建修复 PR
```
- 说明问题现象
- 说明根因分析
- 说明修复方案

## 检查清单

- [ ] 分支创建（/branch）
- [ ] 问题诊断完成（/debug）
- [ ] 测试用例编写（/test）
- [ ] Bug 修复完成
- [ ] 测试通过
- [ ] 提交信息规范（/commit）
- [ ] PR 创建（/pr）
